# 再レンダリング（パート2）
- なぜ再読み込みもしていないのに値が変わっているかというと
- Reactのコンポーネントが再レンダリングされているから
- つまり閉じたコンポーネントの中で初期値を読み込んで値を読み込んで、画面がレンダリングされます
- 何かステートの更新があった場合は、Reactが検知して、コンポーネントの上から読み込む
- 親コンポーネントが再レンダリングされると、子も再レンダリングされる
- コンポーネント内でものすごい数の再レンダリングが起きるようなコードになっている
- ステートが変更されると1から戻ってしまう。(ステート変更,setOn)
- すでにtrueの場合は、記載しない、
- 一つ目の変数がfalseの場合だけ右側の処理を行う。

# useEffect
- 扱う変数が増えたり、ステートの数が増えてきたりするとそれどれの処理が邪魔しあってうまく動かなくなったりする
- そういう時は関心の分岐、numの値の時のみにしよしてほしい
- そんな時にuseEffectを使用する、()の中に関数を取ります
- useEffectの第二引数に配列を取ります、空の配列だったり、変数を指定することができる
- useEffectの第二引数に空の値を入れるとそのコンポーネントで最初の一回だけに通したい処理を実行できる
- 第二引数の配列の中に変数を設定するとその変数が変化した場合にのみ走らせることができる
- ある変数の状態に応じて何か処理をしていきたい時にuseEffectを使用する、
- この変数の変化だけを見ておきたい時に使用する
